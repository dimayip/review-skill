# Review Skill（复盘 Skill）

自适应智能复盘技能，适用于各种 AI 助手和 LLM 智能体。它使用多个经过验证的框架（KPT、GRAI、4F、KISS、STAR、AAR），根据用户的意图自动选择最合适的框架，引导用户完成结构化的复盘。此技能可适配多种 AI 编程助手和智能体框架。

## ✨ 功能特性

- **🤖 自适应框架选择**：根据用户输入自动选择最合适的复盘框架
- **🔠 选择题驱动交互**：参考 `learn-x` skill 的设计理念，使用 A/B/C/D 结构化选择代替开放式提问，降低认知负荷
- **📋 支持多种框架**：
  - **KPT**（Keep / Problem / Try）- 用于每日快速复盘
  - **GRAI**（Goal / Result / Analysis / Insight）- 用于项目/深度复盘
  - **4F**（Facts / Feelings / Findings / Future）- 用于学习/备考复盘
  - **KISS**（Keep / Improve / Start / Stop）- 用于周/月复盘
  - **STAR**（Situation / Task / Action / Result）- 用于面试/绩效复盘
  - **AAR**（After Action Review）- 用于项目节点/团队协作复盘
- **📝 Markdown 输出**：自动将复盘记录保存为 Markdown 文件
- **🩺 诊断优先**：开始前先问 3 个短问题，理解用户的上下文
- **🔄 定期锁定**：每 3 个维度做一次小结，确保理解一致
- **😈 Devil's Advocate**：对用户的正确答案适度挑战，确保深度而非表面理解

## 🚀 使用方法

### 触发词

当用户说以下任一词语时，AI 会自动加载此 Skill：
- "复盘"、"反思"、"总结"
- "今天做了什么"、"项目进展如何"
- "备考总结"、"学习复盘"
- "帮我复盘"、"我想复盘"

### 示例

#### 示例 1：每日复盘（KPT）
```
用户："帮我复盘一下今天"

AI："好的，我们来复盘。先问你 3 个短问题，帮我理解你的情况：
     问题 1/3：你今天复盘，最想达成什么？
     - A. 梳理一下今天做了什么（单纯记录）
     - B. 找出问题，下次做得更好（改进导向）
     - C. 沉淀经验，形成可复用的方法（经验沉淀）
     - D. 以上都不是，其实是：_____"

用户："B"

AI："问题 2/3：复盘今天，你的情绪基调是？
     - A. 满意/有成就感（进展顺利）
     - B. 焦虑/有压力（遇到阻碍）
     - ..."

（继续使用 KPT 框架，使用选择题驱动提问）
```

#### 示例 2：项目复盘（GRAI）
```
用户："帮我复盘一下 Alpha 项目"

AI："我们用 GRAI 框架来深度复盘 Alpha 项目。先问 3 个诊断问题..."
```

## 🎯 框架选择逻辑

| 用户意图 | 诊断结果 | 推荐框架 |
|---------|---------|---------|
| "今天做了什么" | 目的=A（单纯记录） | KPT（轻量版） |
| "项目进展" | 目的=B（改进导向） | GRAI + AAR |
| "备考总结" | 目的=C（经验沉淀） | 4F |
| "本周/本月总结" | 目的=任意 | KISS |
| "面试复盘" | 目的=任意 | STAR |
| 复杂情绪 | 目的=任意 | 4F（先处理情绪） |

## 🛠️ 安装方法

本技能可适配多种 AI 助手和智能体框架。安装方法取决于你的平台：

### 用于 CodeBuddy IDE

运行以下命令安装该 Skill：

```bash
npx skills add https://github.com/dimayip/review-skill.git
```

### 用于其他 AI 助手
1. 将 `SKILL.md` 文件内容复制到你的助手的系统提示词或自定义指令中。
2. 根据你的助手环境调整文件路径和命令示例。
3. 核心逻辑（自适应框架选择、选择题驱动交互）可在任何基于 LLM 的助手中复用。

## 💡 设计理念

本 skill 参考 `learn-x` skill 的核心理念：
> **用结构化选择（A/B/C/D）代替开放式提问。**
> 选择题降低认知负荷，逼用户做 reasoning，而不是空洞回答。

### 核心原则

1. **诊断优先** - 先问 3 个短问题，建立用户的"起始画像"
2. **一次一个维度** - 每次只问一个复盘维度，不堆砌问题
3. **首选选择题** - 能用 A/B/C/D 的，不用开放式提问
4. **留白** - 永远保留"以上都不是，其实是 D: ..."选项，鼓励用户跳出框架思考
5. **定期锁定** - 每 3 个维度，做一次小结，确保理解一致
6. **Devil's Advocate** - 用户回答后，适度挑战，确保深度而非表面
